# HU-310 — Billing Avanzado + Historial de Pagos
**Sprint:** 3 | **Rama:** B | **Media**

> **NOTA:** La integración de pagos con Transbank ya se implementó en Sprint 2 (HU-204 a HU-209, Rama A).
> Esta HU ahora se enfoca en funcionalidades avanzadas de billing.

## Definición
Implementar historial de pagos, lógica de upgrade/downgrade entre planes, y endpoint de estado de transacciones.

## Tareas

### 1. API historial de pagos
`GET /api/billing/transactions`
- Retorna `billing_transactions` del usuario, ordenadas por fecha desc
- Incluye: buy_order, plan_type, amount, status, created_at

### 2. Lógica de upgrade/downgrade
- Si usuario tiene plan Pro_Mensual activo y compra Pro_Anual:
  - Calcular días restantes del plan mensual
  - Aplicar como crédito al plan anual (o simplemente extender desde hoy)
- Si usuario tiene plan Pro_Anual y compra Pro_Mensual:
  - No permitir downgrade mientras el anual esté activo

### 3. UI historial en /facturacion
- Tabla con últimas 10 transacciones
- Estado con badge (aprobado/rechazado/cancelado)
- Monto formateado en CLP

## Archivos
- `src/app/api/billing/transactions/route.ts` (nuevo)
- `src/app/(dashboard)/facturacion/page.tsx` (agregar sección historial)

## Criterios de aceptación
- [ ] GET /api/billing/transactions retorna historial del usuario
- [ ] Lógica upgrade Pro_Mensual → Pro_Anual funciona
- [ ] Downgrade bloqueado con mensaje explicativo
- [ ] UI muestra historial con estados
