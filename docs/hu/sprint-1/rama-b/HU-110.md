# HU-110 — Motor de Liquidación v3.1
**Sprint:** 1 | **Rama:** B | **CRÍTICA**

## Historia
Como sistema quiero un motor de cálculo correcto de liquidaciones TCP chilenas.

## REGLA IUSC (Art. 42 N°1 DL 824)
**Base = RLI = imponible_ajustado − AFP − Salud − APV_B. NUNCA sobre bruto.**

## Definición
Implementar calcularLiquidacion() de Sección 4.2 de GESTI_V3.1_ESPECIFICACION_DEFINITIVA.md:

### Algoritmo (11 pasos)
1. Tasa AFP trabajador
2. Sueldo base (bisección si líquido + impuesto)
3. Ajustar inasistencias + horas extra
4. descuento_afp = imponible × tasaAFP
5. descuento_salud = imponible × 0.07
6. APV Régimen B: MIN(apv, UF×50) — reduce RLI
7. **RLI = imponible − AFP − Salud − APV_B**
8. **IUSC = calcularImpuestoRenta(RLI)** ← sobre RLI
9. Sueldo líquido = imponible − AFP − Salud − IUSC − APV_B
10. Total pago = líquido + no_imponibles − anticipo
11. Cotizaciones empleador (existentes + 4 reforma)

### Bisección (resolverBrutoDesdeNeto)
```
Para cada mid:
  afp = mid × tasaAFP
  salud = mid × 0.07
  rli = mid − afp − salud       ← CORRECTO (no mid directo)
  iusc = IUSC(rli)
  neto = mid − afp − salud − iusc
```

### Reglas adicionales
- Cesantía TCP trabajador = 0% siempre
- Reforma: 4 cotizaciones nuevas, exentos si pensionado/>65
- RIMA para licencia parcial (campo input)
- motor_version = "v3.1"
