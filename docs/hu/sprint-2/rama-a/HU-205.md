# HU-205 — API Checkout con Transbank Webpay Plus
**Sprint:** 2 | **Rama:** A | **Alta**

## Definición
Reescribir `POST /api/billing/checkout` para usar Transbank Webpay Plus en vez de Stripe.

## Flujo
1. Autenticar usuario (JWT)
2. Recibir `{ plan: 'Pro_Mensual' | 'Pro_Anual' }`
3. Generar `buyOrder` único: `GESTI-{plan}-{userId.slice(0,8)}-{timestamp}`
4. Calcular monto desde `PLAN_PRICES[plan]`
5. Llamar `webpayTransaction.create(buyOrder, sessionId, amount, returnUrl)`
   - `sessionId` = `user.id`
   - `returnUrl` = `{NEXT_PUBLIC_APP_URL}/api/billing/callback`
6. Guardar en Supabase tabla temporal o en metadata del buyOrder:
   - `user_id`, `plan_type`, `buy_order`, `amount`
   - Usar tabla `billing_transactions` o campo JSONB en `user_billing`
7. Retornar `{ success: true, data: { url, token } }`
8. Frontend redirige: `window.location.href = url + '?token_ws=' + token`

## Tabla auxiliar (opcional pero recomendada)
```sql
CREATE TABLE IF NOT EXISTS billing_transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES user_profiles(id),
  buy_order TEXT NOT NULL UNIQUE,
  plan_type TEXT NOT NULL,
  amount INTEGER NOT NULL,
  status TEXT DEFAULT 'pending',  -- pending | approved | rejected | cancelled
  tbk_response JSONB,
  created_at TIMESTAMP DEFAULT now()
);
```
Esto permite trackear el estado de cada intento de pago.
