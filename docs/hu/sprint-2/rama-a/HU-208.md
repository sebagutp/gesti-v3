# HU-208 — UI Facturación con Transbank
**Sprint:** 2 | **Rama:** A | **Media**

## Definición
Actualizar la página `/facturacion` para que funcione con Transbank en vez de Stripe.

## Cambios respecto a versión Stripe

### Eliminar
- Botón "Gestionar suscripción en Stripe"
- Referencia a `has_subscription`
- Propiedad `stripePlan` en el array de planes

### Agregar
- Botón "Renovar plan" cuando `plan_status === 'expired'` o `days_remaining < 7`
- Mensaje de estado: "Tu plan vence en X días" con color según urgencia
- Manejo de query params de retorno:
  - `?status=cancelled` → toast "Pago cancelado"
  - `?status=rejected` → toast "Pago rechazado"
  - `?status=timeout` → toast "Tiempo agotado"

### Flujo de checkout
1. Click "Contratar" o "Renovar"
2. `POST /api/billing/checkout` con `{ plan }`
3. Recibe `{ url, token }`
4. `window.location.href = url + '?token_ws=' + token`
   (O según documentación de transbank-sdk, puede ser form POST)

### Success page
- `/facturacion/success` — mostrar confirmación
- No necesita `session_id` de Stripe — verificación ya ocurrió en callback

## Archivos
- `src/app/(dashboard)/facturacion/page.tsx` (modificar)
- `src/app/(dashboard)/facturacion/success/page.tsx` (simplificar)

## Criterios de aceptación
- [ ] Sin referencias a Stripe en la UI
- [ ] Botón "Contratar" redirige a Webpay
- [ ] Botón "Renovar" aparece cuando plan expirado/por vencer
- [ ] Query params de retorno muestran mensajes apropiados
- [ ] Success page funciona sin session_id
